返回了三个结果

使用

这里就简单指出几个我使用的，具体可以查询gmvault documentation

首先是完整的备份。

然后是指定标签备份，这个更有用处。

‘in:tags’ 中的tags就是标签，默认会备份在$HOME/gmvault-db目录下

接下来说一下转换成mbox文件吧

这样就可以把备份下来的右键转换成mbox文件保存了，不然的话邮件默认是eml文件形式的，我就不说第一次备份下来硬盘多了N多文件的事情了。

转换完，突然想到mbox格式打开查询太不方便了，于是呢，就想到把刚才备份的右键全部保存到另外一个gmail里去算了，方便查询。

Windows 08 R2 Virtualbox 64位系统

2015-07-17
vm, windows, 服务器

上周又和老板去买了一台E5的服务器..

配的是超微的双路主板x10drl-i.. 32G内存标配..
无盘和一些小东西做完之后闲的无聊.. 于是果断开虚拟机安装linux然后做远程..

反正配置够用..

先试用了的微软自家的Hyper-v虚拟技术.. 怎么说呢.. 对Arch这种发行版的支持实在是.. 差的不行.. 连正常的关机都做不到应该没有集成包..

微软果然吭.. 安装了Virtualbox之后看不到64位系统的安装选项.. 第一反应是bios的vt-x和vt-d没有开.. 重启进bios都是开着的.. =_=#

PS:为什么服务器的板子开机自检时间这么长… 好长好长的说..

开机一看还是没有.. 想了一下.. 是不是虚拟技术被hyper-v用掉了.. 果断在角色里把hyper-v删除掉.. 重启..

世界清静了..

最后我总结了一下要在windows下用virtualbox安装64位系统有三个条件

1、 vt-x 要打开

2、 vt-d要打开

3、 hyper-v一定要关掉

3、 hyper-v一定要关掉

3、 hyper-v一定要关掉

重要的话要说三遍….
