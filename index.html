Linux NTU Citrix访问

发表于 2017-01-28   |   分类于 Linux   |  

前言

学校是提供了Xen Windows的远程访问的，起初的目的是给使用OSX的学生提供方便。

然而，在外网环境下，我的访问始终失败。

过程

在学校上网，我是一直挂着VPN的，这个举动直接导致我被排除在学校内网之外。由于经常需要在外办公，并且我也不是很喜欢把VPN关掉。

把这个问题解决也就排上日程了。

首先报错是某证书没有被信任，没有被选择。

第一步，我到远程地址的登陆界面，通过Chrome的地址栏把所有证书下载。复制到证书路径。

当然失败了。

第二部，开了一个Windows虚拟机并且成功登陆一次远程之后，通过MMC导出相关证书。并且复制到Linux的证书路径。

结果还是失败了。

结果

在Citrix的论坛某贴，看到有人说通过Firefox导出证书复制，成功了。

SSL ERROR 61 : You have not chosen to trust the issuer…

11楼的回复。

我按照提示做了相关操作之后成功了。

至此，我终于可以在外网环境在我的Linux平台使用学校的Xen远程服务了。

iDrac更新固件

发表于 2017-01-04   |   分类于 服务器   |  

前言

服务器的远程Console用不了，打给Dell客服之后提示更新iDrac固件。更新完之后，内网打开处于空白界面，什么都看不到，外网访问正常。

过程

不是很明白为什么会导致这个问题，通过外网登陆iDRAC Web界面之后在Web Server设定中看到一个TLS选项。改成TLS1.0或者更高之后问题解决。

结果

Windows Server 2008 R2 系统中使用的TLS协议应该是过低了，Google了一番之后确实找到了一个补丁。Update to enable TLS 1.1 and TLS 1.2 as a default secure protocols in WinHTTP in Windows

我直接在Web Server当中把TLS协议要求调低了。
