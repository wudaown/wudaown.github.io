Server 08 R2 禁用 IPv6

发表于 2016-08-10   |   分类于 服务器   |  

前言

服务器装好DNS服务器之后发现dns进程的内存占用超级高！高到不正常。看了一下dns.exe进程的内存占用基本到了12-16GB。

过程

看进程的名字就知道和DNS服务有关了，因为要做AD域的关系DNS服务必须开启。上网转了一圈之后发现这个问题还挺普遍，仔细一看竟然是IPv6的问题的。

dns.exe high memory usage in Server 2008 R2

参考这篇文章，作者还是花了很多时间去了解的。

微软也有想过的KB929852

结果

如果服务器是多线程的，并且IPv6不用的话，还是禁用掉吧

那剩下就比较简单了，就是修改端口了。
