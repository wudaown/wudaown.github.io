Server 2008 R2文件备份杂记

前言

两周前公司某员工收到一封伪装成Fedex的邮件，里面包含nemucod病毒的JS代码。因为所有文件都是储存在服务器上面的，直接导致了改员工账户所有可写文件被加密。此处省略一万字，总之文件是恢复过来了。上面也发话了，只好把准备年底在做的备份提前了。

思路

因为公司所有文件都是在服务器上面的，所有备份量会比较大。并且之前也用过群辉的NAS，这次也就直接考虑入手群辉的NAS了。经过一番调查，DSM6.0支持经过btfs加持的Snapshot Replication和Hyper Backup。

两种方案都很好，深入阅读之后发现两种技术都是建立在对硬盘的区块进行还原的，也就是说并不适用于服务器上面的文件系统。不过我觉得问题不大，明年改其他电脑的时候顺便把资料移动到群辉上面再用上述方案吧。现在的话先用Windows Server Backup问题不大，把备份放在NAS上面就可以了。事实证明调查的不够彻底，被微软坑了，下面会详细说明。

在看介绍的时候顺便留意到了Snapshot Replication可以和Windows系统中的以前的版本结合。不是很清楚是什么原理，不过我记下来了，就顺便再服务器上面把卷影副本打开了。

准备

  1. DS916+

  2. 文件服务器

  3. 网络设备

    其实就是买了一台DS916+和两块4TB的西数红盘，稳定性第一嘛。

实施

我并不在公司，让公司的人帮我装上硬盘，把NAS接入到交换机就可以了。第一个问题，根据官方的手册 http://find.synology.com/ 并找不到NAS。只好下载了群辉的Setup Assistant，发现了NAS的IP地址是169.254.x.x。瞬间就明白了，内网有线网络我并没有DHCP服务器，所有台式电脑都是静态IP的，到路由器把DHCP打开立马分配到了IP。登陆到NAS只好第一件事情就是分配静态IP。

第二件事情就是加入域，都是比较直白的内容了。

比较坑的就是Windows Server Backup的计划备份任务不支持在远程位置保存超过一份的备份内容，也就是说每次备份只好之前的备份就会消失。

我这里已经是生产环境了，就不截图了。但是我坚持要每天保存一份的话应该怎么办呢。Google了好久都没有找到答案，只有一些人提议每次都做一份完整备份就可以保存多个不同时期的备份了，这样太浪费空间了。

刚好看到备份出来的文件是一个大包了的VHD文件，让我想到了和不建立一个虚拟硬盘挂载到本机作为备份用硬盘呢。

diskmgmt.msc

打开磁盘管理创建VHD，创建完成后会自动挂载的不用担心，手动初始化。这里强烈推荐创建的时候选择动态扩展，不然会很慢很慢的。

图中BACKUP DISK就是我的备份虚拟硬盘啦，实际使用下来并没有什么问题除了性能会有所下降之外。

关于卷影副本，我的理解就是Mac的TM嘛。不知道为什么我某个分区默认有一个VSS副本了，要换位置的时候还不能删除旧的。

vssadmin delete shadowstorage /for=G: /on=G: 也是失败的，Google了一圈之后发现可以尝试一下diskshadow

diskshadow

list shadows all 这一段的输出我没看明白

delete shadows oldest G:

这样就正常了，在员工的计算机上面也可以恢复了。

当我以为一切都结束继续检查日志的时候，看到这样一条日志。

好吧，VSS服务默认是手动开启的。系统经过一段时间会自动关闭服务，这个不是打问题，改成手动的就可以了。

最终效果是这样的。

后记

其实对于虚拟硬盘并且在备份的时候选择VSS完整备份我并不是很放心，先观察几天有可能会改成VSS副本备份。

文件量相差的还是很大的。
